- subscriptionStartAt (timestamp/date)
- subscriptionEndAt (timestamp/date)
- manualSuspended (boolean) OR isSuspended (boolean) with suspendedReason

- suspendedReason (string nullable)
- features (json) containing per-restaurant feature flags (pos, qr, etc.)

3) Auto Suspend logic MUST be enforced server-side:
   - A restaurant is considered "expired" if now >= subscriptionEndAt.
   - Expired restaurants must be blocked from:
     - restaurant login

     - public QR ordering endpoints

   - Super admin can "Suspend now" anytime (sets manualSuspended/isSuspended true)
