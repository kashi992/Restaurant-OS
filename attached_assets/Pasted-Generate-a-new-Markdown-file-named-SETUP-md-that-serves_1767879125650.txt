3. Local Development Setup (macOS / Linux / Windows)
Prerequisites
Node.js v20+ (node -v)
PostgreSQL 14+ (psql --version)
npm or yarn
Optional: openssl (to generate secrets)
Step-by-Step
Clone the repo
npm install
cp .env.example .env → edit with your values
Create DB:

# macOS/Linux (with pg CLI)  
createdb restaurant_pos  
# Windows (via pgAdmin or psql): CREATE DATABASE restaurant_pos;  
Push schema: npm run db:push
Seed data (optional but recommended): npx tsx scripts/seed.ts
Start: npm run dev → open http://localhost:5000
Windows-Specific Notes
Use PowerShell or Git Bash (not CMD) for cp, openssl
If createdb fails, connect via psql and run CREATE DATABASE restaurant_pos;
Ensure PostgreSQL service is running

7. Architecture Notes for Developers
The backend (server/) serves the frontend in production
In dev, Vite runs on :5173, Express proxies / → Vite
All API routes are prefixed with /api
Multi-tenancy: every query must filter by restaurant_id (handled via middleware later)
Shared types in shared/schema.ts keep frontend/backend in sync
8. Troubleshooting
“ECONNREFUSED”: DB not running or wrong DATABASE_URL
Blank screen on Replit: Check browser console; often CORS or 404 from wrong base path
Drizzle push fails: Ensure DB user has CREATE privileges
Port already in use: On local, change PORT in .env; on Replit, restart repl
Seeding fails: Run db:push first
9. Going to Production (Preview)
Set NODE_ENV=production
Use strong secrets (never reuse dev ones)
Always use HTTPS
Reverse proxy (nginx) recommended
Monitor memory/CPU (Socket.IO scales per tenant)
