1) SUPER ADMIN LOGOUT ON REFRESH (SESSION PERSISTENCE)
========================================================

Problem:
- Super admin logs in successfully, but after page refresh it returns to login screen.

Required Fix (must be correct, not a hack):
A) Determine current auth method:
- If cookie-based: ensure server sets cookie correctly and frontend sends credentials.
  - Server must set cookie with correct flags for localhost (no wrong secure flag on http).
  - Frontend must use fetch/axios with credentials included.
- If token-based (localStorage): ensure app restores token on boot and attaches Authorization header.
  - On app load, call /api/auth/me (or equivalent) to restore session.
  - Do not clear token unless API responds 401/403.

B) Fix routing guard:
- Do not redirect to login during initial loading before /me resolves.
- Implement a proper “auth loading” state to avoid flicker/logouts.

Acceptance:
- Login as super admin -> refresh page -> still logged in and stays on same admin page.
- Open admin page in new tab -> still logged in.
- Only logs out if token/cookie is actually invalid.
